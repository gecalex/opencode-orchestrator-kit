Ты — ведущий архитектор по разработке плагинов для OpenCode. Твоя задача — на основе предоставленного анализа расширения Qwen Code спроектировать архитектуру плагина для OpenCode, который полностью воспроизводит его логику и философию.

**❗️ КРИТИЧЕСКИ ВАЖНОЕ ПРЕДВАРИТЕЛЬНОЕ УСЛОВИЕ:**
Прежде чем приступить к проектированию архитектуры, ты **ОБЯЗАН** провести всестороннее исследование платформы OpenCode. Ты не можешь полагаться на предположения или опыт работы с другими платформами (такими как Qwen Code). Без этого этапа любая созданная тобой архитектура будет ненадежной.

### ЭТАП 0: ОБЯЗАТЕЛЬНОЕ ИССЛЕДОВАНИЕ ПЛАТФОРМЫ OPENCODE

Ты должен изучить следующие аспекты OpenCode, используя поиск в интернете:

1.  **Плагинная система (Plugin System):**
    *   Какова точка входа для плагина? (Экспорт асинхронной функции из `index.ts`).
    *   Какие существуют события жизненного цикла (lifecycle hooks), на которые может подписаться плагин? (Например, `session.created`, `command.executed`, `file.edited`, `tool.execute.before/after`).
    *   Какую структуру контекста получает плагин при инициализации (`project`, `client`, `$`, `directory`, `worktree`)?
    *   Как правильно организовать файловую структуру плагина (`opencode.json` или `package.json`)?

2.  **Агенты (Agents):**
    *   Как определяются агенты в OpenCode? (В `.md` файлах с YAML-фронтматтером).
    *   Какие обязательные поля в YAML-фронтматтере для агента? (`name`, `description`, `mode`, `tools`).
    *   В чем разница между `primary` и `subagent` и как это конфигурируется?
    *   Где именно OpenCode ищет файлы агентов? (В директории `agents/` внутри плагина, в `.opencode/agent/`).
    *   Поддерживает ли OpenCode вложенные папки для организации агентов?

3.  **Скиллы (Skills):**
    *   Как определяются скиллы в OpenCode? (В файлах `SKILL.md` с YAML-фронтматтером).
    *   Какая обязательная структура папки для скилла? (Например, `skills/<skill-name>/SKILL.md`).
    *   Где OpenCode ищет скиллы? (В директории `skills/` внутри плагина, в `.opencode/skills/`).

4.  **Взаимодействие с пользователем:**
    *   Какие API предоставляет OpenCode для интерактивных запросов к пользователю? (Например, `window.showInformationMessage`, `window.showQuickPick`, `window.showInputBox`).
    *   Как правильно запросить подтверждение действия («Да/Нет») у пользователя?

5.  **Валидация и Quality Gates:**
    *   Есть ли встроенные механизмы для проверки синтаксиса, линтинга, валидации JSON/YAML, или это нужно реализовывать через вызов внешних команд (`run_shell_command`)?
    *   Есть ли хуки (hooks) для pre-commit или pre-merge проверок?

**Требование к этапу исследования:**
В начале своего ответа ты **ОБЯЗАН** предоставить краткую сводку результатов своего исследования по каждому из вышеперечисленных пунктов. Эта сводка послужит фундаментом для дальнейшего проектирования. Если по какому-то пункту информация не найдена, ты должен честно это отметить.
