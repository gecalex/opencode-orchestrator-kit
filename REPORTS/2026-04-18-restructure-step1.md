## 📋 Summary
Выполнена реструктуризация проекта для разделения исходного кода плагина (разработка) от runtime-файлов, необходимых для работы плагина в внешних проектах.

## 🔧 Changes Made
- Переместил директорию `src/` из `.opencode/src` в корень проекта (`./src`).
- Переместил директорию `node_modules` из `.opencode/node_modules` в корень проекта (`./node_modules`).
- Обновил `tsconfig.json`:
  - Изменил `outDir` с `"./dist"` на `"./.opencode/dist"` чтобы скомпилированный плагин помещался в runtime-папку.
  - Оставил `rootDir` как `"./src"` (исходный код теперь в корне).
- Обновил `opencode.json`:
  - Раскомментировал и исправил путь к плагину на `"file://.opencode/dist/index.js"`.
- Выполнил сборку плагина через `npm run build` (или `tsc`), которая успешно создала файлы в `.opencode/dist/`.
- Проверил, что в `.opencode/dist/` присутствуют скомпилированные JavaScript-файлы (index.js и другие).
- Убедился, что больше нет ссылок на старый путь `.opencode/src` в кодовой базе.

## ✅ Results
- Исходный код плагина и зависимости разработки теперь находятся в корне проекта, что соответствует стандартам разработки (источник truth).
- Директория `.opencode/` содержит только runtime-часть: определения агентов (`agents/`), скиллов (`skills/`), скомпилированный плагин (`dist/`) и вспомогательные скрипты (`scripts/`).
- Плагин по-прежнему корректно собирается и указывает на правильный входной пункт через `opencode.json`.
- Структура проекта теперь четко разделяет:
  - **Корень проекта** — файлы разработки плагина (исходный код, конфиги, документация, отчёты).
  - **Внутренняя директория `.opencode/`** — то, что отдаётся опенкоду как готовый плагин.

## ⚠️ Issues & Risks
- Требуется убедиться, что скрипты в `.opencode/scripts/` (например, `validate-yaml.js`) по-прежнему работают正确 после перемещения `node_modules` в корень (они используют относительные импорты и должны быть в порядке, так как node_modules теперь на уровень выше).
- При дальнейшей разработке необходимо помнить о запуске `npm run build` после изменений в исходном коде для обновления `.opencode/dist/index.js`.
