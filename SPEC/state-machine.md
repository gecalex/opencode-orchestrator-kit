# Спецификация модуля state-machine.ts

## 1. Обзор

Модуль state-machine.ts управляет состоянием проекта в рамках методологии Speckit. Он определяет 10 состояний проекта от 0 до 9, контролирует переходы между состояниями и обеспечивает соблюдение правил workflow.

## 2. Функциональность

- Отслеживание текущего состояния проекта (0-9)
- Валидация переходов между состояниями
- Определение разрешенных агентов для каждого состояния
- Предоставление информации о следующем возможном переходе
- Блокировка переходов, пока не выполнены все условия текущего состояния
- Поддержка отката к предыдущим состояниям при критических ошибках

## 3. Интерфейсы

### Входы:
- Текущее состояние проекта (из storage или инициализации)
- Запрос на переход к определенному состоянию
- Информация о выполненных условиях текущего состояния
- Подтверждение пользователя на критических переходах

### Выходы:
- Текущее состояние проекта
- Список разрешенных агентов для текущего состояния
- Информация о следующем возможном переходе
- Статус валидности запроса на переход
- Описание текущего состояния

### Внутренние методы:
- `getCurrentState(): StateCode` - возвращает текущее состояние
- `canTransitionTo(targetState: StateCode): boolean` - проверяет возможность перехода
- `getAllowedAgents(stateCode: StateCode): AgentType[]` - возвращает разрешенных агентов
- `getNextState(stateCode: StateCode): StateCode | null` - возвращает следующее состояние
- `validateTransition(currentState: StateCode, targetState: StateCode): ValidationResult` - валидирует переход
- `getStateDescription(stateCode: StateCode): string` - возвращает описание состояния

## 4. Ограничения

- Переходы между состояниями линейны (0→1→2→...→9) без пропусков
- Каждое состояние блокирует переход к следующему до выполнения всех условий
- Критические переходы требуют явного подтверждения пользователя
- Откат возможен только к предыдущему состоянию (rollback)
- Состояние 9 (Релиз-готов) является финальным перед релизом

## 5. Внешние интеграции

- Используется модулем `blocking-rules.ts` для проверки пре-условий
- Интегрируется с `orchestrator.ts` для определения разрешенных агентов
- Используется модулем `session-hooks.ts` для отслеживания событий сессии
- Предоставляет данные модулю `reporting.ts` для генерации отчетов о прогрессе

## 6. Требования

1. Должен поддерживать ровно 10 состояний проекта (0-9)
2. Каждое состояние должно иметь четкое описание и критерии выполнения
3. Должен блокировать недопустимые переходы между состояниями
4. Должен предоставлять информацию о разрешенных агентах для каждого состояния
5. Должен поддерживать проверку готовности к переходу
6. Должен возвращать описание состояния для отображения пользователю
7. Должен быть независим от конкретной реализации хранилища состояния
8. Должен обеспечивать потокобезопасность при concurrent доступе
