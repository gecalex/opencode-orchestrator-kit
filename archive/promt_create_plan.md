### 2. Иерархическая структура агентов (КРИТИЧЕСКИ ВАЖНО)

**Никакой плоской структуры агентов!** Агенты должны быть организованы по следующей схеме:
agents/
├── {domain}/ # домен: devops, backend, frontend, testing, speckit, orchestrator, health, security и т.д.
│ ├── orchestrator/ # тип: оркестраторы (только для координации, не пишут код)
│ │ └── {имя_агента}.md
│ └── worker/ # тип: воркеры (пишут код, тесты)
│ └── {имя_агента}.md

**Пример:**
- `agents/backend/worker/python_developer.md`
- `agents/testing/orchestrator/tdd_coordinator.md`
- `agents/speckit/constitution-agent.md`

**Правила именования папок:**
- `domain` — область ответственности (backend, frontend, testing, devops, speckit, orchestrator, security, health).
- `type` — `orchestrator` или `worker`.

Ты должен спроектировать эту иерархию в рамках плана и создать соответствующие директории и файлы.

### 3. Процесс выполнения задач и Git Workflow

После утверждения плана (или сразу после его представления) ты приступишь к **последовательному** выполнению задач.

**Для каждой задачи:**
1. **Создай отдельную feature-ветку** из `develop` с именем `feature/T-XXX-краткое-описание`.
2. **Реализуй задачу** (напиши код, создай файлы агентов/скиллов).
3. **Выполни проверку результата**:
   - Для TypeScript-кода — убедись, что нет синтаксических ошибок (можно использовать `tsc --noEmit`).
   - Для `.md` файлов — проверь корректность YAML-фронтматтера (валидный YAML).
   - Для общей логики — убедись, что задача полностью выполнена согласно спецификации.
4. **Если проверка успешна**:
   - Выполни `git add .`
   - Сделай коммит с сообщением в формате [Conventional Commits](https://www.conventionalcommits.org/): `type(scope): описание`. Например: `feat(core): реализовать машину состояний проекта`.
5. **Вернись в `develop` и влей feature-ветку** с помощью `git merge --no-ff feature/T-XXX-...`.
6. **Сообщи пользователю** о завершении задачи и переходе к следующей.

**Если проверка НЕ успешна:**
- Сообщи об ошибке, исправь её в той же feature-ветке, повтори проверку и только потом делай коммит и вливай.

### 4. Язык и стиль

- Код должен быть чистым, хорошо структурированным, с использованием TypeScript.
