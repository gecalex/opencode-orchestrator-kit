# Подробный план реалирации OpenCode Orchestrator Kit в соответствии с Qwen Orchestrator Kit

**Дата:** 2026-04-18  
**Статус:** В РАЗРАБОТКЕ

## Цель
Реализовать полноценную систему оркестрации разработки на основе методологии Speckit/TDD с динамической адаптацией под стек технологий, метапрограммированием агентов и обязательным TDD workflow, соответствующую философии и логике оригинального Qwen Orchestrator Kit.

## Текущее состояние
Базовый каркас плагина реализован:
- Машина состояний с 4 состояниями (10/20/30/40)
- Pre-Flight проверки (10 пунктов)
- Quality Gates (3 из 5 точек реализованы)
- Базовое разделение агентов
- Инструментальные ограничения на основе состояний
- Система промптов и отчетов
- Базовая документация

## Необходимые компоненты для полной реализации

### 1. MCP Dynamic Resolution System
- Автоматический поиск MCP серверов по технологиям из ТЗ
- Установка и настройка найденных серверов
- Интеграция с workflow перед фазой реализации
- Кеширование и повторное использование настроек

### 2. Метапрограммирование агентов
- Механизм создания отсутствующих агентов (work_dev_meta_agent)
- Система futures-агентов и их автоматического создания
- Проверка структуры созданных агентов
- Динамическая загрузка агентов без перезапуска сессий

### 3. Расширенная система workflow (12 фаз)
- Фаза 1: Анализ состояния проекта
- Фаза 2: Создание конституции проекта
- Фаза 3: Создание спецификаций модулей
- Фаза 4: Анализ противоречий в спецификациях
- Фаза 5: Планирование реализации (ОБЩИЙ план)
- Фаза 6: Разбивка на задачи с зависимостями
- Фаза 7: Анализ пробелов в требованиях
- Фаза 8: Назначение агентов на задачи
- Фаза 9: Создание TEST/CODE разделения (TDD First!)
- Фаза 10: TDD Координация (языковая специализация тестировщиков)
- Фаза 11: Разработка кода под тесты
- Фаза 12: Выполнение задач и интеграция

### 4. Обязательное TDD First! с TEST/CODE разделением
- Принудительное создание тестовых задач перед кодовыми
- Изолированные сессии для написания тестов
- Проверка покрытия тестов (≥80%) перед переходом к коду
- Языково-специализированные тестовые агенты
- WORKFLOW: TEST → RED → CODE → GREEN → Quality Gate

### 5. Блокирующие правила с автоматическими проверками
- Аварийный тормоз перед запуском любых агентов
- Проверка пре-условий (чек-лист) перед каждой операцией
- Автоматическая блокировка при нарушении правил
- Обязательное утверждение пользователя на критических переходах
- Интерактивные диалоги подтверждения

### 6. Детальная система отчетности
- Структурированные отчеты о делегировании в markdown формате
- Отчеты после каждой операции с результатами и следующими шагами
- Журнал выполнения сессий
- Дашборд прогресса проекта
- Система отслеживания нарушений границ агентов

### 7. Принцип минимальных привилегий в полном объеме
- Матрица разрешений для каждого типа агента:
  - Оркестраторы: только task tool, нет write_file
  - Планировщики/аналитики: чтение файлов, поиск, анализ
  - Рецензенты: чтение файлов, комментарии, нет редактирования кода
  - Кодеры: запись/редактирование файлов, базовые инструменты
  - Тестировщики: запись тестовых файлов, запуск тестов
  - Специалисты: узкоспециализированные инструменты для их домена
- Запрет прямого доступа к опасным операциям для неподходящих агентов

### 8. Комплексная документация
- Файл AGENTS.md как основная документация для проектно-специфических инструкций (аналогично QWEN.md в оригинале)
- Разделы: блокирующие правила, быстрый старт, паттерн выполнения задач, таблицы состояний, чек-листы пре-условий, workflow, отчеты после операций, git workflow, quality gates детали, speckit workflow детали, MCP dynamic resolution, сохранение контекста, приложения с примерами команд и типичными ошибками
- Поддержка ссылок на внешние файлы через @синтаксис

## План реализации по этапам

### Этап 1: Базовый каркас и инфраструктура (2-3 недели)
1.1. Улучшение существующей машины состояний
1.2. Реализация системы хуков для событий сессий
1.3. Создание базовой системы кастомных инструментов
1.4. Реализация механизма сохранения контекста между сессиями
1.5. Обновление документации и AGENTS.md

### Этап 2: Расширенный workflow и многоагентность (3-4 недели)
2.1. Реализация 12-фазовой системы workflow с табличным FSM
2.2. Добавление системы специализированных агентов (планировщик, кодер, рецензент, тестировщик и т.д.)
2.3. Реализация изолированных сессий рецензирования для проверки артефактов
2.4. Внедрение базовой системы TDD с разделением TEST/CODE
2.5. Система утверждений пользователя на критических переходах workflow
2.6. Расширение Quality Gates до всех 5 точек

### Этап 3: Продвинутые функции и интеграция (4-6 недель)
3.1. Реализация MCP Dynamic Resolution System
3.2. Внедрение метапрограммирования агентов (work_dev_meta_agent)
3.3. Детализация блокирующих правил с автоматическими пре-проверками
3.4. Разработка комплексной системы отчетности
3.5. Углубление принципа минимальных привилегий до полной матрицы разрешений
3.6. Создание полной документации AGENTS.md
3.7. Интеграция всех компонентов и end-to-end тестирование

## Критерии готовности
- Все 12 фаз workflow реализованы и протестированы
- MCP Dynamic Resolution System функционирует
- Метапрограммирование агентов работает
- Обязательное TDD First! enforceтся
- Блокирующие правила предотвращают нарушения workflow
- Система отчетности предоставляет детальную обратную связь
- Принцип минимальных привилегий соблюден для всех типов агентов
- Документация AGENTS.md completa и соответствует оригиналу (по дублированию функционала QWEN.md)
- Плагин проходит интеграционное тестирование в реальных сценариях использования

## Риски и mitigation strategies
1. **Сложность MCP интеграции** - начать с базовой реализации для популярных технологий, постепенно расширять
2. **Ограничения метапрограммирования агентов** - использовать символические ссылки и перезагрузку конфигурации вместо горячей замены
3. **Производительность при большом количестве фаз** - оптимизировать проверки состояния и кэшировать результаты
4. **Сопротивление изменению рабочего процесса** - сделать переход плавным с обратной совместимостью к базовому функционалу
5. **Сложность тестирования end-to-end workflow** - разработать комплексные сценарии тестирования для каждой фазы

## Следующие шаги
Начать с Этапа 1: реализовать улучшенную машину состояния и систему хуков, затем перейти к Этапу 2.
