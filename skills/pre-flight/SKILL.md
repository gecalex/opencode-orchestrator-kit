## When to Use

Используй этот скилл:

- ✅ Перед началом любой фазы разработки
- ✅ После инициализации проекта
- ✅ Перед запуском Speckit workflow

НЕ используй для:

- ❌ Ежедневных проверок (используй health-команды)
- ❌ Проверки кода (используй code-quality-checker)

## Инструкции

### Шаг 1: Проверка Git репозитория

```bash
git rev-parse --git-dir || echo "ERROR: Git не инициализирован"
```

### Шаг 2: Проверка ветки develop

```bash
git branch --list | grep develop || echo "ERROR: Ветка develop не найдена"
```

### Шаг 3: Проверка .gitignore

```bash
test -f .gitignore || echo "ERROR: .gitignore не найден"
```

### Шаг 4: Проверка конституции

```bash
test -f .opencode/specify/memory/constitution.md || echo "ERROR: Конституция не найдена"
```

### Шаг 5: Проверка агентов

```bash
ls .opencode/agents/*.md | wc -l
```

### Шаг 6: Проверка скиллов

```bash
ls .opencode/skills/*/SKILL.md | wc -l
```

## Результат

- Все проверки пройдены → можно продолжать разработку
- Есть ошибки → остановиться и исправить перед продолжением

## Блокирующий

**Да**, этот скилл блокирующий — останавливает процесс при ошибках.
