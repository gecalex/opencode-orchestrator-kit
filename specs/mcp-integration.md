# Спецификация модуля mcp-integration

## 1. Обзор

Модуль mcp-integration обеспечивает динамическое разрешение и управление Model Context Protocol (MCP) серверами в Orchestrator Kit. Он состоит из двух основных компонентов: mcp-registry.ts (реестр MCP серверов) и mcp-resolution.ts (система разрешения MCP серверов по требованиям задачи).

## 2. Функциональность

### 2.1 MCP Registry (mcp-registry.ts)
- Хранение реестра известных MCP серверов (20+ серверов в 12 категориях)
- Предоставление информации о доступных MCP серверах
- Категоризация MCP серверов по типам (базы данных, поиск, файлы/Git, автоматизация, DevOps, память, коммуникация)
- Поддержка версионирования MCP серверов
- Обновление реестра MCP серверов

### 2.2 MCP Resolution (mcp-resolution.ts)
- Анализ требований задачи для определения необходимых технологий
- Поиск подходящих MCP серверов в реестре по ключевым словам и технологиям
- Автоматическая установка и настройка недостающих MCP серверов
- Кеширование успешно настроенных MCP серверов для повторного использования
- Валидация работоспособности установленных MCP серверов
- Управление жизненным циклом MCP серверов (обновление, откат)

## 3. Интерфейсы

### Входы для MCP Registry:
- Запросы на поиск MCP серверов по технологии, категории или ключевым словам
- Запросы на получение информации о конкретном MCP сервере
- Команды на обновление реестра MCP серверов

### Выходы из MCP Registry:
- Список MCP серверов, соответствующих критериям поиска
- Детальная информация о конкретном MCP сервере (ID, название, описание, репутация, benchmark score, доступные версии)
- Статус успешности операций с реестром

### Входы для MCP Resolution:
- Требования задачи (из спецификаций модулей, описания задач)
- Текущий стек проекта (из конституции и анализа состояния)
- Список уже установленных и доступных MCP серверов
- Запросы на установку конкретного MCP сервера

### Выходы из MCP Resolution:
- Определенные необходимые технологии для задачи
- Список рекомендованных MCP серверов для установки
- Статус установки и настройки MCP серверов
- Кеш настроенных MCP серверов с метками времени
- Результаты валидации работоспособности MCP серверов

### Взаимодействие между компонентами:
- MCP Resolution использует MCP Registry для поиска доступных серверов
- MCP Resolution может обновлять кеш и информацию в реестре о успешно установленных серверах
- Оба компонента взаимодействуют с оркестратором для получения команд и отправки результатов

## 4. Ограничения

- MCP Registry должен поддерживать минимум 20 MCP серверов в 12 категориях
- MCP Resolution должен анализировать требования задачи и определять необходимые технологии
- Установка MCP серверов должна выполняться автоматически без вмешательства пользователя (за исключением критических ошибок)
- Кеш MCP серверов должен иметь механизм обновления и очистки устаревших записей
- Все операции с MCP серверами должны проходить проверку работоспособности (валидацию)
- MCP Resolution должен блокировать переход к фазе реализации, если критические MCP серверы недоступны (Quality Gate 6)
- Должна быть обеспечена изолированная установка MCP серверов (не конфликтующая с глобальными установками)
- Должна поддерживаться откат к предыдущей версии MCP сервера при проблемах с новой версией

## 5. Внешние интеграции

- Взаимодействует с оркестратором (orchestrator.ts) для получения команд и отправки результатов
- Интегрируется с state-machine.ts для проверки разрешенности операций в текущем состоянии проекта
- Используется модулем quality-gates.ts (Gate 6: MCP Check) для проверки доступности MCP серверов перед фазой реализации
- Взаимодействует с agent-system для назначения задач mcp-manager агенту
- Интегрируется с session-management для обработки событий сессии, связанных с MCP серверами
- Использует файловые операции для кеширования конфигурации MCP серверов (.opencode/mcp-cache/)
- Взаимодействует с внешними реестрами и репозиториями MCP серверов (npm, PyPI, crates.io и т.д.)
- Интегрируется с системой разрешений (permissions.ts) для проверки прав на установку и настройку MCP серверов

## 6. Требования

1. MCP Registry должен предоставлять актуальную информацию о минимум 20 MCP серверах в 12 категориях
2. MCP Resolution должен автоматически определять необходимые технологии из спецификаций задач и текущего стека проекта
3. MCP Resolution должен осуществлять поиск подходящих MCP серверов в реестре по ключевым словам и технологиям
4. MCP Resolution должен автоматически устанавливать и настраивать недостающие MCP серверы
5. MCP Resolution должен кешировать успешно настроенные MCP серверы для повторного использования с механизмом обновления
6. MCP Resolution должен проводить валидацию работоспособности установленных MCP серверов
7. MCP Integration должен блокировать переход к фазе реализации, если критические MCP серверы (filesystem, git, memory) недоступны
8. MCP Integration должен поддерживать откат к предыдущей версии MCP сервера при обнаружении проблем с новой версией
9. Все операции с MCP серверами должны выполняться в изолированной среде без влияния на глобальную установку
10. MCP Integration должен предоставлять четкие интерфейсы для взаимодействия с другими модулями системы
11. MCP Integration должен обрабатывать ошибки при установке и настройке MCP серверов и предлагать решения
12. MCP Integration должен поддерживать обновление реестра MCP серверов из внешних источников
